SLIDE 02

프롬프트의 3가지 유형

목적에 따라 프롬프트 작성 방식이 달라집니다

유형 1 서술형

맥락과 배경을 설명하여 AI가 상황을 이해하도록 유도하는 방식

예시
당신은 10년 경력의 무역 전문가입니다. (페르소나) 중소기업 수출 담당자가 HS 코드를 처음 배우는 상황입니다. 이 사람에게 HS Code의 개념과 중요성을 설명해 주세요.
유형 2 지침형

단계별 절차나 명확한 규칙을 제시하여 원하는 출력을 유도하는 방식

예시
다음 규칙에 따라 파이썬 코드를 작성하세요. 1. 함수명은 snake_case로 작성 2. 각 함수의 docstring(함수 안의 주석)을 포함 3. 예외 처리 반드시 포함 (try와 except) 4. 결과는 json 형식으로 반환
유형 3 함수형

입력값과 출력값을 명확히 정의하여 일관된 결과를 생성하는 방식

예시
입력 : {product_name}, {hs_code}, {country} 처리 : 해당 국가의 관세율 및 수입 규제 조회 출력 : { "관세율" : "", "규제사항" : [], "필요서류" : [] }

방식 정의 특징

퓨샷 예시 2개 이상 — HS Code 분류 예시
HS Code를 분류해 주세요 예시) 품목 : "면으로 만든 남성용 티셔츠" → HS Code 품목 : "폴리에스테르 소재 여성 블라우스" → HS Code 품목 : "아동용 데님 바지" → HS Code 품목 : "울 혼방 남성 재킷" → HS Code

비평가 페르소나

다음 3명의 전문가 입장에서 우리 제품의 동남아 수출 전략을 분석해 주세요. 김부장 (영업 15년, 낙관적, 현장 중심) 이박사 (시장 조사 전문, 데이터 중심, 신중함) 박대표 (스타트업 창업가, 속도 중심, 리스크 감수) 각자의 관점에서 찬반 의견을 제시하세요.

패턴 유형 1 단순 나열형
# 지시 문제를 해결하는 방법을 최소 4가지 제시하세요. ## 방법 {번호} : {이름} - 핵심 원리 : {한 줄} - 적합한 상황 : {언제 쓰면 좋은가} - 단점 : {주의할 점} - 난이도 : {초급 / 중급 / 고급} ## 문제 수출 통관 서류를 자동으로 생성하고 싶습니다.
패턴 유형 2 축 기반 비교형
아래 4가지 방법을 기준으로 비교 분석하세요. 평가 기준 : - 개발 속도 (빠름 / 보통 / 느림) - 유지 보수 난이도 (쉬움 / 보통 / 어려움) - 비용 (무료 / 저비용 / 고비용) 비교 대상 : FastAPI, Django, Flask, Node.js Express 용도 : 무역 데이터 처리 REST API 서버
패턴 유형 3 상황별 분기형
아래 상황별 가장 적합한 데이터베이스를 추천하세요. 각 상황은 독립적으로 판단하세요. 상황 A : 스타트업, 개발자 1명, 빠른 MVP 상황 B : 무역 데이터 10년치, 복잡한 통계 쿼리 상황 C : 실시간 환율·가격 데이터, 초당 1,000건 업데이트 상황 D : 문서 형태의 비정형 통관 데이터 저장 각 상황에 대해 : [DB명], [이유], [주의사항]
패턴 유형 4 역발상 대안형
아래 문제에 대해 일반적인 해결책뿐만 아니라 "하지 않는" 선택지와 "문제 자체를 재정의"하는 방법도 대안으로 포함하여 총 5가지를 제시하세요. 문제 : 수입 통관 처리 시 시간이 너무 오래 걸린다.

"블록체인이 무역 금융에 어떻게 활용되나요?" — 이용자에 따른 차이
이용자답변 방향
무역을 처음 배우는 대학생개념부터 쉽게, 비유 활용, 친근한 어투
중소기업 수출 담당 팀장실무 적용 방법, 절차, 비용 중심
금융 기관의 IT 의사 결정자기술 스택, 보안, ROI, 구현 복잡도 중심
실전 예시 중소기업 대표를 위한 이용자 페르소나
# 이용자 정보 - 직책 : 중소기업 대표 (제조업, 직원 30명) - 수출경험 : 국내 영업만 10년, 해외 수출 처음 - 관심사 : 리스크 최소화, 빠른 실행 - 우려 : "복잡하고 어려울 것 같다"는 막연한 두려움 - 시간 : 바빠서 긴 글을 읽지 않음 # 지시 위의 이용자에게 첫 수출 준비 3단계를 설명하세요. 전문 용어를 최소화하고, 각 단계는 3줄 이내로 작성하세요.
이용자 페르소나 설계 체크리스트

레시피 패턴

목표 달성을 위한 단계별 작업 절차를 체계적으로 받는 방법

핵심 원리 : 다음에 무엇을 해야 할지 고민하는 시간을 줄이자 — 일반 질문 대신 단계별 레시피를 요청하면 실행 가능한 로드맵을 얻을 수 있습니다.
기본 STEP 형식
## STEP {번호} : {단계명} - 할 일 : {구체적 행동} - 소요 시간 : {예상 기간} - 완료 기준 : {이것이 되면 다음으로} - 주의 사항 : {흔한 실수}
실전 예시 Python 초보자의 HS Code 자동 분류 프로그램 만들기
# 목표 Python 초보자가 무역 HS Code 자동 분류 프로그램 만들기 # 현재 상태 - Python 설치만 되어 있음 - 프로그래밍 경험 없음 - 엑셀로 품목 데이터 500개 보유 # 요청 위의 목표를 달성하기 위한 단계별 레시피를 작성하세요. 각 단계는 아래 형식으로 작성하세요. ## STEP {번호} : {단계명} - 할 일 : {구체적 행동} - 소요 시간 : {예상 기간} - 완료 기준 : {이것이 되면 다음으로} - 주의 사항 : {흔한 실수}
분기 포함 레시피 상황에 따라 다른 다음 단계
무역 계약 클레임 처리 절차를 단계별로 작성하세요. 단, 각 단계에서 발생할 수 있는 분기 상황도 포함하세요. 형식 : STEP N. {행동} - 정상 : {이 결과면 다음 단계로} - 이상 : {이 결과면 어떻게 대응}
체크리스트 형식 실행 체크리스트로 받기
신규 수출 거래처와의 첫 계약 체결을 위한 실행 체크리스트를 작성하세요. 형식 : [ ] {행동} — {이유 또는 주의 사항}

SLIDE 11

뒤집힌 상호작용 패턴

AI가 먼저 묻고, 사용자가 답하면서 완성도를 높이는 방식

방법 1 질문 먼저, 답은 나중
제가 원하는 결과물은 {목표}입니다. 바로 만들지 말고, 먼저 필요한 정보를 질문해 주세요. 모든 정보가 수집된 후 결과물을 작성하세요.
방법 2 한 번에 하나씩
제가 원하는 결과물은 {목표}입니다. 한 번에 하나의 질문만 하세요. 제가 답하면 다음 질문으로 넘어가고 충분한 정보가 모이면 결과물을 작성하세요.
실전 예시 1 수출 제안서 작성
수출 제안서를 작성해 주세요. 바로 작성하지 말고, 먼저 아래 항목들을 질문해서 제 답변을 모두 수집한 후 작성하세요. 수집이 필요한 정보 : - 제품 정보 - 타깃 바이어 - 핵심 강점 - 가격 정책 - 납기 조건
실전 예시 2 Python 학습 계획서
Python 학습 계획서를 만들어 주세요. 단, 한 번에 하나의 질문만 하고 제 답변을 들은 뒤 다음 질문으로 넘어가세요. 5~7개 질문이 끝나면 맞춤형 학습 계획서를 작성해 주세요.
이 패턴의 장점 : AI가 임의로 가정하지 않고, 실제 상황에 맞는 정보를 수집한 후 결과물을 생성하므로 완성도가 크게 높아집니다.
SLIDE 12

인지 검증자 패턴

복잡한 문제를 하위 질문으로 분해하여 단계적으로 검증한 후 통합 판단을 내리는 방식

인지 검증자 패턴 처리 흐름
① 복잡한 질문 입력
→
② AI가 하위 질문 분해
→
③ 각 질문 순서대로 검증
→
④ 검증된 답 통합
복잡한 질문 예시
"우리 회사가 베트남에 생산 법인을 설립하면 세금, 인력, 물류 측면에서 유리한가요?"
이런 복잡한 질문은 그냥 던지면 AI가 표면적인 답만 합니다. 인지 검증자 패턴을 적용하면 하위 문제를 먼저 분해하여 훨씬 깊은 분석이 가능합니다.
방법 1 명시적 분해 요청
아래 질문에 바로 답하지 말고, 먼저 이 질문을 정확히 답하기 위해 필요한 질문을 3~5개 도출하세요. 각 하위 질문에 순서대로 답한 뒤 최종 통합 답변을 제시하세요. 질문 : Python으로 무역 데이터 자동화를 구축할 때 클라우드 서버가 필요한가요?
방법 2 전문가 검증 방식
{전문 분야} 전문가처럼 이 문제를 다루세요. 확신이 없는 부분은 가정으로 명시하고, 각 판단의 근거를 함께 제시하면서 단계적으로 결론에 도달하세요.
이 패턴의 핵심 : 복잡한 문제는 한 번에 답하기 어렵습니다. AI가 스스로 문제를 쪼개고, 각 조각을 검증한 뒤 통합하도록 유도하면 훨씬 신뢰할 수 있는 결론을 얻을 수 있습니다.
